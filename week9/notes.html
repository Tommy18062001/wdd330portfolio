Chapter 9: Window Object

The window object represents the browser window that contains a web page.

The Browser Object Model:

BOM for short is a collection of properties and methods that contain information about the browser and computer screen.

  • Going Global: Global variables are actual properties of a global object. In a browser environment, the global object is the window object.
  • Dialogs: There are few types of dialogs like: alert(), confirm(), and prompt(). They are also window objects.

Browser Information:

The window object has a number of properties and methods that provide information about the user’s browser.

Which Browser?

window.navigator.userAgent – This property will return information about the browser and operating system being used.

Location, Location, Location

The window.location property is an object that contains information about the URL of the current page.

  • window.location.href: return the full URL as a string
  • window.location.protocol: return the protocol used (such as https, pop2, ftp etc.)
  • window.location.host: returns the domain of the current URL and the port number
  • window.location.port: returns the port number
  • window.location.pathname: returns a string of the path that follows the domain
  • window.location.search: returns a string that starts with a “?” followed by the query string parameters
  • window.location.hash: returns a string that starts with a “#” followed by the fragment identifiers
  • window.location.origin: returns a string that shows the protocol and domain where the current page originated from

Some methods of the window.location object:

  • window.location.reload(): Force a reload of the current page.
  • window.location.assign(): Load another resource from a URL provided as a parameter
  • window.location.replace(): Same as the window.location.assign() method, except that the current page will not be stored in the session history, so the user will be unable to navigate back to it using the back button.

The Browser History

The window.history property can be used to access information about any previously visited pages in the current browser session.

  • window.history.lenght: shows how many pages have been visited before arriving at the current page.
  • window.history.go(): used to go to a specific page, where 0 is the current page
  • window.history.forward(): navigate forwards by one page.
  • window.history.back(): backwards by one page respectively

Controlling Windows

  • window.open(): open a new window. It takes URL as a parameter with few arguments. Example: window.open(‘https://sitepoint.com’,‘SitePoint’,‘width=400, ↵height=400,resizable=yes’)
  • window.close(): In the other hand, this method is used to close a window
  • window.move(): move a window depending on the argument given (X, Y)
  • window.resizeTo(): Resize window’s dimensions by taking two parameters that specify the width and height

Screen Information

The window.screen object contains information about the screen that the browser is displayed on.

  • window.screen.height
  • window.screen.width
  • window.screen.availWidth
  • window.screen.availHeight
  • window.screen.colorDepth: used to find the color bit depth of the user’s monitor

The Document Object

Each window object contains a document object. This object has properties and methods that deal with the page that has been loaded into the window.

  • document.write(): writes a string of text to the page. This is an archaic method of writing text to the document and should be avoided.
  • document.cookie: create a cookie

Cookies:

Cookies are small files that are saved locally on a user’s computer. They are also limited to storing up to 4KB of data, although 20 cookies are allowed per domain, which can add up to quite a lot of data. Their use for data storage is starting to be replaced in many cases by the new HTML5 localStorage API as it allows more data to be stored The method used to deal with cookies is document.cookie

Timing Functions

The window object provides some useful methods for scheduling the execution of a function and for repeatedly executing functions at regular intervals

  • window.setTimeout() method accepts a callback to a function as its first parameter and a number of milliseconds as its second parameter
  • window.clearTimeout()
  • window.setInterval() method works in a similar way to setTimeout except that it will continue to invoke the callback function after every given number of milliseconds.
  • window.clearInterval()

Animation

The setTimeOut() and setInterval() methods can be used to animate elements on a web page. But after, a new method has been developed.

  • window.requestAnimationFrame(): this works in much the same way as the window.setInterval() method, although it has a number of improvements to optimize its performance.
  • window.cancelAnimationFrame()

Chapter 14: HTML5 APIs

HTML5 is the latest iteration of the Hypertext Markup Language used to create web pages. This version goes beyond the actual markup language, however, and brings together a number of related technologies such as CSS and JavaScript.

The data- Attribute:

It is a way of embedding data in a web page using custom attributes that are ignored by the browser. They’re also private to a page, so shouldn’t be used by an external service: their sole purpose is to be used by a JavaScript program.

The names of these attributes can be decided by the developer, but they must use the following format:

  • start with data
  • contain only lowercase letters
  • include an optional string value

If a data- attribute’s name contains hyphens, they are replaced with camel-case notation, so data-max-length would be accessed using maxLength.

Data attributes provide a convenient way of adding data directly into the HTML markup, enabling a richer user experience.

HTML5 APIs

The HTML5 specification contains a number of APIs that help to gain access to the hardware, such as cameras, batteries, geolocation, and the graphics card. Hardware evolves quickly and APIs are frequently introduced to give developers access and control new features that appear in the latest devices. Due to the ever-changing nature of most APIs, it is still best practice to use object-detection tests before using any of the API methods.

HTML5 Web Storage

The Web Storage API provides a key-value store on the client’s computer that is similar to using cookies but has fewer restrictions, more storage capacity, and is generally easier to use.

The Web Storage API has some crucial differences with cookies:

  • information stored is not shared with the server on every request
  • information is available in multiple windows of the browser (but only if the domain is the same)
  • storage capacity limit is much larger than the 4KB limit for cookies

If a browser supports the Web Storage API the window object will have a property called localStorage, which is a native object with a number of properties and methods used to store data.

Every time a value is saved to local storage, a storage event is fired. The event object sent by the event listener to the callback has a number of properties that provide information about the updated item:

  • key tells us the key of the item that changed
  • newValue tells us the new value to which it has been changed
  • oldValue tells us the previous value before it was changed
  • storageArea tells us if it is stored in local or session storage

Geolocation

The Geolocation API is used to obtain the geographical position of the device.

This has a method. Here are some example

  • getCurrentPosition() that will return a position object to a specified function, called youAreHere()
  • clearWatch() method can be used to stop the callback being called, using the ID of the watch as an argument

There are many other properties:

  • position.speed: returns the ground speed of the device in meters per second
  • position.altitude: returns an estimate of the device’s altitude in meters above the WGS84 ellipsoid, which is a standard measurement for the center of the Earth
  • position.headin: returns the direction the device is moving in as a bearing
  • position.timestamp property returns the time that the position information was recorded
  • position.accuracy property returns the accuracy of the latitude and longitude properties in meters

Web Workers

We saw in earlier chapters that JavaScript is a single-threaded language, meaning that only one process can run at one time. Web workers allow processes to be run in the background, adding support for concurrency in JavaScript.

To get started, we need to use the Worker() constructor function to create a new worker

Web workers use the concept of messages to communicate back and forth between the main script and worker script. (use of the method postMessage())

When a worker has completed its task, it can be stopped using the terminate() method or close () inside the script

Shared Web Workers

The examples we have seen so far are known as dedicated web workers. These are linked to the script that loaded the worker and are unable to be used by another script. You can also create shared web workers that allow lots of different scripts on the same domain to access the same worker object.

They are a useful feature that help to keep sites responsive, even when complicated operations are being carried out.

Multimedia

Before HTML5 it was notoriously difficult to display audio and video in browsers, and plugins such as Flash often had to be used. HTML5 introduced the and tags used to insert audio and video clips into a web page. It also introduced a Media API for controlling the playback of the clips using JavaScript.

Audio and video elements have a number of properties and methods to control the playback of the clip.

  • play() method will start the clip playing from its current position
  • pause() method will pause the clip at its current position
  • volume property is a number that can be used to set the audio volume
  • muted property is a Boolean value that can be used to mute the audio
  • currentTime property is a number value that can be used to jump to another part of the clip
  • playbackRate property is used to fast-forward or rewind the clip by changing its value. A value of 1 is playback at normal speed
  • loop property is a Boolean value that can be set to true to make the clip repeat in a loop
  • duration property can be used to see how long the clip lasts

Other APIs

The list of APIs is constantly growing, and include APIs for accessing a device’s camera, using WebSockets for multiple simultaneous connections, uploading files, accessing the battery status, handling push notifications, building drag and drop functionality, creating 3D effects with WebGL, and many more! A comprehensive list of HTML5 APIs can be found at the Mozilla Developer Network.

Drawing with Canvas

The canvas element was introduced to allow graphics to be drawn onto a web page in real time using JavaScript.

A canvas element is a rectangular element on the web page. It has a coordinate system that starts at (0,0) in the top-left corner. To add a canvas element to a page, the tag is used specifying a height and width.

Method and properties:

  • getContext() method is used to access the context
  • The fill and stroke colors can be changed by assigning a CSS color to the fillStyle and strokeStyle properties respectively
  • fillRect() method is used to draw a filled-in rectangle.
  • strokeRect() method works in the same way, but produces a rectangle that is not filled in
  • moveTo() and lineTo() is used to draw
  • stroke(): nothing will be displayed on the screen unless this method is called
  • arc() method is used to draw an arc of a given radius from a particular point
  • fillText() method is used to write text onto the canvas

Shims and Polyfills

HTML5 APIs progress at a rapid rate―new APIs are constantly being introduced and existing APIs often change. Modern browsers are very quick to update and implement many of the changes, but you can’t always guarantee that users will have the most up-to date browser. This is where a shim or a polyfill comes in handy. These are libraries of code that allow you to use the APIs as usual; they then fill in the necessary code that not provided natively by the user’s browser.

  • Shim: is a piece of code that adds some missing functionality to a browser, although the implementation method may be differ slightly from the standard API.
  • polyfill is a shim that achieves the same functionality while also using the API commands that would be used if the feature was supported natively.
